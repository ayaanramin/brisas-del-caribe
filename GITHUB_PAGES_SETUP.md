# GitHub Pages Deployment Guide

This is a static Vite website optimized for GitHub Pages deployment.

## Build & Deploy

### Local Build
```bash
pnpm install
pnpm run build
```

The build output will be in the `dist/` folder.

### Deploy to GitHub Pages

#### Option 1: Using GitHub Actions (Recommended)

1. Create `.github/workflows/deploy.yml`:
```yaml
name: Deploy to GitHub Pages

on:
  push:
    branches:
      - main

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v2
        with:
          version: 10
      - uses: actions/setup-node@v4
        with:
          node-version: '22'
          cache: 'pnpm'
      - run: pnpm install
      - run: pnpm run build
      - uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./dist
```

2. Push to GitHub and enable GitHub Pages in repository settings:
   - Go to Settings → Pages
   - Source: Deploy from a branch
   - Branch: gh-pages, folder: / (root)

#### Option 2: Manual Deployment

1. Build locally: `pnpm run build`
2. Push the `dist/` folder to a `gh-pages` branch
3. Enable GitHub Pages in repository settings

## Project Structure

- `client/` - React application source code
- `client/src/` - Components, pages, and styles
- `client/index.html` - Entry HTML file
- `vite.config.ts` - Vite configuration
- `dist/` - Build output (generated by `pnpm run build`)

## Configuration

The Vite configuration is set to:
- Root: `client/` folder
- Output: `dist/` folder
- Target: Static HTML/CSS/JS

No backend or server code is included. This is a pure frontend application.
